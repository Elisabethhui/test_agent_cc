# Repository Overview: context_system

## What the Repository Does

This is a **multi-agent RAG (Retrieval-Augmented Generation) system** with the following core capabilities:

1. **Agent Orchestration**: Multiple specialized agents (Plan, Explore, General, Verification, Guide, Statusline, Coordinator) that work together to complete complex tasks.

2. **Context Compression**: Automated context management with:
   - **Micro-compaction**: Cleaning up redundant tool outputs
   - **AI-based summarization**: Using LLM to compress conversation history when approaching token limits

3. **Tool System**: Built-in tools for file operations (Read, Write, Edit), file search (Glob, Grep), and shell execution (Bash) with permission-based approval.

4. **Session Persistence**: Auto-save conversation history with `/resume` capability for continuing interrupted sessions.

5. **Sandbox**: Bubblewrap-based isolation for running bash commands safely.

6. **Companion System**: An AI pet/buddy in the terminal with stats, mood, and interactive gameplay.

7. **KAIROS Memory**: Cross-session memory consolidation system.

## Which Files to Read First

**Primary entry points (in order of priority):**

1. **`main.py`** - Entry point, shows overall flow
2. **`config.py`** - Configuration with token thresholds and tool mappings
3. **`olmx_client.py`** - LLM client implementation
4. **`agents/__init__.py`** - Agent registry and initialization
5. **`core/session_memory.py`** - Singleton memory management

**Core runtime files:**

1. **`core/executor.py`** - Agent execution loop
2. **`core/compact_system.py`** - AI-based context compression
3. **`core/micro_compact.py`** - Context cleanup logic

**cc-mini entry points:**

1. **`cc-mini/src/core/main.py`** - REPL entry point
2. **`cc-mini/src/core/engine.py`** - Core streaming API loop
3. **`cc-mini/src/core/compact.py`** - Compression logic
4. **`cc-mini/src/core/llm.py`** - LLM client wrapper
